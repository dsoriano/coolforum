1 . Configuration nécessaire
==============================

	Pour pouvoir utiliser ce forum sur votre site, il faudra que votre hébergeur
puissent exécuter des scripts PHP 4 et possède un serveur de base de données MySQL.
Vous aurez en outre besoin de certains codes pour son installation : l'adresse
de votre serveur MySQL, votre login et votre mot de passe, rien de plus. Si vous ne
les connaissez pas, ces renseignements vous sont fournis par votre hébergeur.

	N'hésitez pas à visiter notre site Web http://www.coolforum.net pour
connaître les meilleurs hébergeurs aptes à supporter votre forum.


2 . Caractéristiques
======================

	CoolForum est ce que l'on appelle un forum de discussion. Il vous permet
d'agrémenter votre site web d'un outil interactif avec vos visiteurs afin de leur
donner la parole et de converser avec eux. Grâce à la base de donnée, tous les
messages sont conservés et vos visiteurs pourront ainsi consulter à n'importe
quel moment les archives des messages.

	CoolForum vous propose de nombreuses options que vous pourrez découvrir à
mesure que vous l'utiliserez. Au fil des versions, il se révèle être un véritable
logiciel d'application internet, simple et intuitif. Aucune connaissance en
programmation n'est requise et son installation sur votre hébergement internet
s'effectue en quelques minutes. Vous découvrirez rapidement qu'il tend à devenir
un véritable forum professionnel grâce à ses nombreuses options et possibilités.


3 . Installation Complète
==========================

	L'installation de CoolForum est trés simple : avant de transférer vos fichiers
sur votre hébergement internet, dans le répertoire "secret", renommez le fichier
"connect.base.php en "connect.php" puis éditez le avec un éditeur de texte comme
le bloc-note par exemple. Vous devez à présent y rajouter quelques informations qui
vous permettront de vous connecter au serveur MySQL de votre hébergeur.

Ce fichier se présente ainsi:

<?php
//------------- CONNECTION A LA BASE -------------------
class My_SQL
{
	var $host	=	"xxxxx"; 		// serveur hôte
	var $user 	= 	"xxxxx";		// nom d'utilisateur
	var $pass 	= 	"xxxxx";		// mot de passe de connexion
	var $bdd 	= 	"xxxxx";		// nom de la base de donnée
}
$_PRE="CF_";
?>

les xxxxx représentent les renseignements à modifier. Ces codes vous sont fournis par votre
hébergeur. Une fois configuré, sauvegardez vos modifications.

Transférez vos fichiers sur votre espace d'hébergement grâce à votre client FTP en
respectant la hiérarchie des dossiers et des fichiers du forum.

Toujours à l'aide de votre logiciel FTP, passez le répertoire "logos" en chmod 777 afin de
donner les droits d'écriture, sinon les logos ne pourront être uploadés.

Note: les comptes sur Free ou Online par exemple ne permettent pas de passer un répertoire
en chmod 777, cependant l'upload de fichiers fonctionne quand même.

Avec votre navigateur, rendez-vous ensuite à l'adresse de votre forum dans le dossier "install"
sur la page "install.php". Choisissez l'option "installation complète" et laissez vous guider.

Une fois le message de fin d'installation affiché, Vous DEVEZ supprimer complètement le
dossier "install" de votre compte et rejoindre l'administration pour configurer votre forum.

Vous pouvez vous rendre dans l'administration en allant dans le répertoire "admin" du
forum. Vous aurez à vous identifier, puis vous aurez accès à l'administration.

	Nous avons finalement préféré vous laisser configurer vos codes de connexions à MySQL
afin d'éviter tout problème d'installation pour ceux qui n'ont pas les autorisations pour
écrire et modifier des fichiers avec les fonctions PHP prévues à cet effet.


4 . Mise à jour de la version 0.8.4 beta à 0.8.5 beta
=====================================================

La mise à jour de la version 0.8.4 à la version 0.8.5 ne contient aucune modification de base
de donnée, seulement des mises à jour de fichiers :
    - Tous les fichiers ".php" à la racine du forum
    - Tous les fichiers ".php" du dossier "admin"
    - Tous les fichiers ".php" du dossier "lng/fr"

Changelog apporté par cette nouvelle version:
---------------------------------------------

	- Nettoyage du code afin d'améliorer la compatibilité avec les version de PHP 5.2, 5.3, 5.4.
	La prise en charge de PHP 5.5 n'a pas été testée.

5 . Conseils
==============
	A Propos des emails!
	--------------------
		Certains hébergeurs ne permettent pas l'envoi d'email en PHP.
	Si vous l'installez sur ce type d'hébergement, n'oubliez pas de désactiver dans
	l'administration les fonctions relatives aux emails.
		Egalement certains autres hébergeurs n'utilisent pas la fonction mail()
	classique mais la fonction email(). Le forum supporte désormais cette fonction chez
	les hébergeurs Online et Nexen, n'oubliez pas de le configurer dans votre 
	administration !!

	Les Nouvelles versions !!!
	--------------------------
		A partir de la version 0.7, vous avez la possibilité de verifier à partir de
	votre forum si une nouvelle version est sortie. Ce système fait une requête http sur
	le site officiel afin de vérifier si une nouvelle version est sortie. Nous 
	garantissons qu'il n'y a aucun spyware et qu'aucune information n'est relevée lors de
	ces requêtes. Cependant, pour ceux qui en douteraient, l'option de vérification de
	nouvelle version ne se fait pas automatiquement et nécessite une action de 
	l'administrateur.

	N'oubliez pas cependant que les nouvelles versions peuvent apporter des corrections
	de bugs et de sécurité importantes, donc ne les négligez pas et maintenez vos forums
	à jour !!!
